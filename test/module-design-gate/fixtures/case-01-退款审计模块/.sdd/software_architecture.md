# 软件架构说明（测试 fixture）

> 本文件是测试 fixture 的一部分，用于 module-design-gate 的模块边界判定。仅为本次测试场景构造，不代表真实系统。

## 模块定义表

| 模块名 | 职责 | 主要目录 |
|---|---|---|
| 支付审计模块 | 记录支付/退款操作的审计日志，提供审计记录查询能力 | `src/payment-audit/` |
| 订单模块 | 管理订单生命周期、订单状态流转 | `src/order/` |
| 支付模块 | 发起支付、退款请求，与第三方网关交互 | `src/payment/` |

## 模块依赖关系

- 支付审计模块 ← 订单模块（订单模块向审计模块写入审计事件）
- 支付审计模块 ← 支付模块（支付模块向审计模块写入审计事件）
- 订单模块和支付模块之间通过订单服务交互

## 关键约束

- 审计模块**不得**直接修改订单或支付模块的内部数据表。
- 跨模块交互必须通过明确的事件/接口契约，不得旁路写入。
